일 시 : 2020년 03월 28일 (토) 오후 2시 ~ 6시 (4시간 진행)
사용 언어 : C++
문제 수 : 5

문제 1 :
 나는 스택을 사용하여 풀었으며 간단한 문제였다. 대략 10분에서 20분정도 걸렸던것 같다.

문제 2:
 문자열 파싱하는 문제였다.
 왜 나는 항상 문제를 풀때 효율성까지 고려하여 풀려고 하는 것 때문에 시간을 엄청 날렸다.. 하.. 결국에는 효율성과 코드의 간결함? 심플함?을 버리고 문제에서 요구하는 결과에 대하여 몇가지로 나누어서 순차적으로 접근하여 문제를 풀어 해결하였다.
같은 값에 대한 처리를 결국 제출했던 것처럼 간단하게 했으면 시간을 1시간은 벌었을 것 같다.

문제 3:
 경우의 수를 사용하면 풀 수 있지 않을까 생각했던 문제. 간단히 보고 넘겼다.

문제 4:
 엄청 큰 수에 대하여 방을 배정하는 문제였다. 문제만 간단히 보고 넘겼다.

문제 5:
 어떻게 풀까 생각하다가 결국 6시가 되어 종료하게 만들어버린 문제.
 각 원소들의 값이 크고, 배열의 크기가 최소 1개 이상 20만개 이하이다. 이 문제도 결국 효율성을 어떻게 하면 땡길까 생각하며 공책에 이것저것 해봤지만 어떤 것을 적용을 해야 할지 파악하지 못했다. 

결국 푼 문제는 1번과 2번... 나름 최근 알고리즘 문제들을 꾸준히 풀어가고 있지만 아직 많이 부족하다는 생각이 들었다. 문제를 보았을때 해당 문제가 원하는 것을 바로 꿰뚫어 보지 못하는 것 같다. 또한 문제를 너무 복잡하게 생각하고, 효율성까지 생각을 하며 풀려고 하니 시간이 배가 걸리고, 나중에 가서는 결국 복잡해진 머리가 터져서 문제를 마무리하지 못한다. 앞으로도 꾸준히 더 다양한 알고리즘 문제들을 학습하며 문제기 원하는 것을 바로 파악할  수 있는 능력을 기르고, 문제를 풀 때 너무 복잡하게 생각하지 않고 풀어나가야겠다.
